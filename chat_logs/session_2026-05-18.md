# Session 2026-05-18 — Backtest Overhaul

## What we fixed

### Problem: Backtest returning 0 trades / terrible results
The signal backtest was returning 0 trades or very poor results. Three debugging sessions revealed:

**Root cause 1 — Stale server:**
The uvicorn process was started before the scoring code changes. It kept running old thresholds (A+≥80, A≥60) while the new code had (A+≥75, A≥55). Fix: kill and restart with `--reload` flag.

**Root cause 2 — Duplicate trades:**
After a loss, `next_start = setup["bar_index"] + 1` caused the loop to find the same FVG zone on bar+1 (because the same historical FVG is still the closest aligned zone). Same entry/stop/target appeared twice.
Fix: Added `used_entries: list` to `_find_killzone_setup`. After a loss, the entry price is added to `used_entries`. Second attempts skip any setup within `min_stop_pts` of an already-traded entry.

**Root cause 3 — No real HTF direction filter:**
`htf_aligned=True` was hardcoded for both long AND short in `get_ict_analysis`. This gave 15pts of HTF credit to every setup regardless of daily trend.
Fix: `run_backtest` now downloads daily bars (`_download(symbol, ..., "1d")`), computes `get_htf_bias(htf_bars)` per day, and passes `htf_direction` to `_find_killzone_setup`. Setups against the daily trend are skipped.

**Root cause 4 — 20pt minimum stop is noise:**
100% of losses hit exactly at the stop level. 9/15 losses hit at exactly 20pts (the minimum). NQ moves 20pts within a single 5-minute candle. These were all noise stop-hunts.
Fix: Raised `min_stop_pts` 20→50 for MNQ, 8→20 for MES, 5→8 for MGC. Also raised stop_buffer: 20→25 (MNQ).

**Root cause 5 — Edge entry (FVG bottom/top):**
Entering at the very edge of the FVG zone is the most vulnerable level — classic stop hunt target. Price briefly pokes through then recovers.
Fix: Changed entry from `fvg["bottom"/"top"]` to `fvg["mid"]` (FVG midpoint) in `_find_killzone_setup`.

## Key architectural decisions

- **1 trade per day vs 2:** Kept 2-attempt limit but the `used_entries` guard means the second attempt must be at a meaningfully different price level. May 12 showed this is valuable — second attempt at different FVG won.
- **HTF "lean" still allowed:** "bullish_lean" and "bearish_lean" pass the filter (not blocked). Only pure opposite direction is blocked. This avoids being too restrictive in transitioning markets.
- **60d removed from UI:** yfinance only provides ~60 days of 5m intraday data. Adding 3-day buffer made 60d request fail silently. Max is 45d.
- **MES/MGC not ready:** ES/Gold have different microstructure, smaller FVG gaps, smoother price action. ICT iFVG parameters calibrated for NQ don't translate directly. Needs separate work.

## Features Built

### 1. Keep-Alive Ping (`App.jsx`)
- Added `KeepAlive` component that pings `/api/health` every 4 minutes
- Prevents Render free-tier backend from sleeping mid-session
- Invisible component, no UI impact

### 2. Daily P&L Tracker (`Overview.jsx`)
- New `DailyPnLCard` component on the Overview grid (5th card)
- Stores P&L in `localStorage`, resets automatically each day
- `DAILY_LIMIT = 1000` ($1k daily loss limit for Topstep Lucid 25k Pro)
- Progress bar: green when profitable, yellow at 50%+ loss, red at 75%+ loss
- +Add form to log individual trades; Reset button
- Date persisted as `{date, pnl}` — date check prevents carryover

### 3. Pre-Trade Checklist (`/checklist`)
- New page: `frontend/src/pages/Checklist.jsx`
- 9 ICT criteria checked against live `/api/ict/analysis` and `/api/ict/advanced` data:
  1. NY Killzone active (9:30–11:30 AM ET)
  2. HTF bias confirmed (daily/4H alignment)
  3. Grade A or A+ setup
  4. Fresh liquidity sweep
  5. Displacement candle present
  6. iFVG identified
  7. DOL (draw on liquidity) clear
  8. Market structure shift confirmed
  9. No high-impact econ events
- Verdict: **STRONG SETUP** (≥85% met) / **MODERATE** (≥65%) / **NO TRADE** (<65%)
- Auto-refreshes every 60s
- Includes ICT entry sequence quick-reference at bottom
- Added to Sidebar as "Pre-Trade Check" with `ClipboardCheck` icon

### 4. Alert System (`AlertSystem.jsx`) — DISABLED
- Built but NOT mounted in App.jsx
- Polls `/api/ict/alert-check` every 30s in killzone, every 2min outside
- Web Audio API 3-note beep, browser Notification API, toast stack
- **Disabled because:** `Notification.requestPermission()` causes stuck popup on user's machine
- File kept for future re-enable (replace auto-request with manual "Enable Alerts" button)

### 5. ICT Alert-Check Endpoint (`backend/routes/ict.py`)
- New `GET /api/ict/alert-check` endpoint
- Fast, returns `{alert, in_killzone, direction, grade, score, sweep_count, ifvg_count}`
- Only runs full analysis during killzone; returns minimal data otherwise

### 6. Volume-Intensity Candles (`Charts.jsx`)
- New "Vol Color" toggle button in chart toolbar
- Calculates p25/p75 volume percentiles from visible data
- Maps volume → color opacity: p75+ = `ff` (full), p25–p75 = `aa`, below p25 = `55`
- Green for bullish candles, red for bearish — same color, different intensity
- Simulates "elephant candle" concept (Coach Dakota) without variable-width candles
- `volColor` state, included in candle data `useEffect` dependency array

### 7. OR 30m Overlay (`Charts.jsx`)
- Added `or30: false` to `DEFAULT_OVERLAYS`
- Draws purple dashed horizontal lines for Opening Range High/Low (9:30–10:00 AM ET)
- Labeled "OR High" and "OR Low" on chart
- Toggle button added to ICT overlays row in chart toolbar
- Only visible on intraday timeframes

### 8. 30-Minute VWAP Backtest Filter (`backend/routes/backtest.py`)
- New helper: `_get_30min_vwap_bias(day_bars, bar_dt)`
  - Computes cumulative VWAP up to `bar_dt`
  - Finds last completed 30-min window
  - Compares period close to VWAP → returns `'bullish'`, `'bearish'`, or `'neutral'`
- Applied in `_find_killzone_setup()` after HTF filter:
  - If vwap_bias conflicts with trade direction → skip the setup
  - Returns `'neutral'` for first 30 min of session → filter skipped (avoids over-filtering)
- Inspired directly by Coach Jay's methodology (observed in transcripts)
